搭建hexo博客及其主题配置

什么是 Hexo？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装node.js

点击进入node.js官网，下载稳定版或最新版进行安装。

安装git

点击这里进入git的下载网站下载git的安装包进行安装。

开始搭建

注册Github账号

点击这里，进入官网进行注册。

创建仓库

登录账号后，在页面的右上方“+”处选择New repository进行仓库的创建。

在仓库名输入框中输入：

Github昵称.github.io

注意一定要与Github昵称对应。

然后点击Create repository即可。

生成添加秘钥

在任意位置点击鼠标右键，选择Git Bash Here，输入：

ssh-keygen -t rsa -C "Github的注册邮箱地址"

按3个回车，待秘钥生成完毕，在C:/Users/用户名/.ssh/会得到两个文件：id_rsa和id_rsa.pub，用带格式的记事本（推荐sublime text）打开id_rsa.pub，Ctrl + a复制里面的所有内容，然后进入https://github.com/settings/ssh：

将复制的内容粘贴到Key的输入框，点击Add SSH key按钮即可。

安装配置hexo

在你喜欢的位置创建一个文件夹存放hexo文件。
接下来我们的操作都将在Git Bash中进行：

  • 定位博客本地放置的路径

    cd 文件夹名

  • 下载安装hexo

    npm install -g hexo-cli

  • 安装好hexo以后，在bash输入：

    hexo

  • 初始化博客

// 建立一个博客文件夹，并初始化博客，<folder>为文件夹的名称。

hexo init <folder>

// 进入博客文件夹，<folder>为文件夹的名称。

cd <folder>

// node.js的命令，根据博客既定的dependencies配置安装所有的依赖包。

npm install
  • 配置博客

用带格式的记事本打开_config.yml（站点配置文件）

  1. 修改网站相关信息zh-Hans

    title: (标题)
    subtitle: (副标题)
    description: (站点描述，可以是你喜欢的一句签名)
    author: (作者)
    language: zh-Hans
    timezone: Asia/Shanghai
    注意：每一项的填写，其:后面都要保留一个空格，下同。

  2. 配置部署
    在sublime text中按ctrl+f，输入deploy，回车，即可找到对应项。

    deploy:
      type: git
      repo:
      branch: master
    其中repo项是之前Github上创建好的仓库的地址，可以通过如下图所示的方式得到：

发表一篇文章

在bash输入：

hexo new "文章标题"

我们可以在本地博客文件夹source->_posts文件夹下看到我们新建的markdown文件。
用Markdown编辑器（我用的是Cmd Markdown）打开文件，输入内容，点右上角的齿轮，导出Markdown到source->_posts文件夹，然后进行本地发布：

hexo server

打开浏览器，输入：

http://localhost:4000/

目前发布的博客只有本机看得到，怎么让其他人看到我们写的博客呢？接下来我们进行博客的部署。
（注意要等出现 $ 再执行下一句命令。）

hexo clean
hexo generate
hexo deploy

若出现错误:

ERROR Deployer not found : git

执行命令:

npm install hexo-deployer-git --save
hexo g
hexo d

若接下来出现错误:

Please tell me who you are.

这个错误是因为当前本地的Git还没配置用户信息，所以执行以下命令:

git config --global user.email "github的注册邮箱"
git config --global user.name "github昵称"
hexo d

此时，会要求输入用户名，接着要求输入密码，密码输入期间不会看到任何字符，然后直接回车即可。

这时候我们的博客已经部署到网上了，我们在浏览器地址栏输入我们的网址即可访问。

github昵称.github.io
